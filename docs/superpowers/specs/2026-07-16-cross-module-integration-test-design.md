# 跨模块联调测试设计

## 目标

在本地运行系统后，验证“项目池 → 派工/修井运行 → A5 → 物料 → 完井 → 数据分析与报表”的数据、状态和权限能够连续闭环，并输出可实施的优化清单。

## 范围

- 项目池审批、运行表生成和承包商派工。
- A5 模拟审核、日报/回调、签名校验、幂等与异常状态。
- 物料模拟同步及计划、出库、到场、使用状态链路。
- 完井登记对运行表、物料闭环的约束。
- 数据分析、统计报表、交付汇总、导出及上报/辖区单位数据权限。

## 测试策略

先执行后端全量回归、前端生产构建和重点接口契约测试，建立可重复的自动化基线。随后用本地启动脚本运行前后端服务，按一条真实业务数据完成闭环操作并记录接口响应、页面状态及统计结果。

## 核心场景

1. 创建并审批项目，派工后确认运行表、承包商运力和项目状态同步。
2. 触发 A5 模拟审核与日报/回调，验证签名、时间窗、重复事件、未知工单和状态转换。
3. 触发物料模拟事件，验证外部事件幂等、数量校验、状态流转与运行表物料汇总。
4. 物料闭环后登记完井，验证完井记录、运行表状态与完成分类统计。
5. 用全局用户、上报单位用户和辖区单位用户分别查询数据分析、统计导出和交付汇总，验证事实数据和明细均在权限范围内。

## 验收标准

- 自动化测试与前端构建全部通过。
- 每个场景的业务状态、接口响应与页面显示一致；重复回调和重复物料事件不重复写入。
- 数据分析、报表及导出总数能回溯到同一批业务记录；非全局用户不显示跨单位数据。
- 问题按 P0（阻断/越权/数据损坏）、P1（核心流程不一致）、P2（体验或可观测性）分级，并附带复现步骤、影响范围和修复建议。

## 非目标

- 不接入真实 A5 或物料供应商生产接口。
- 不改变已确认的业务状态机，只记录联调发现并在后续优化计划中安排修复。
