# A5、物料与数据分析闭环加固设计

## 1. 背景

当前系统已经具备 A5 日报同步、物料需求管理和统计分析能力，但上线前审查发现以下缺口：

- A5 回调签名校验按大小写读取请求头，无法兼容 ASGI 对请求头名称的小写化处理。
- 未配置 `A5_API_SECRET` 时，任意非空签名都会被接受，生产环境存在伪造回调风险。
- 物料列表、详情、更新、删除和统计没有统一应用当前用户的数据范围。
- 统计看板与 Excel、Word 导出没有显式接收用户范围，可能聚合用户无权查看的数据。
- A5 未配置真实地址时，异常与工序接口只返回空集合，不能提供稳定、可核对的验收样本。
- 物料模块没有外部系统适配边界，无法在真实仓储或供应接口接入前验证完整联动。

本设计采用分层闭环改造：先封闭安全和权限风险，再建立可替换适配器和确定性模拟数据，最后统一统计口径并完成端到端验收。

## 2. 目标与非目标

### 2.1 目标

- A5 回调能够正确识别大小写不同的请求头，并在生产环境缺少安全配置时失败关闭。
- A5 回调具备时间戳校验、防重放、幂等处理和可审计结果。
- 所有物料操作、物料统计、综合统计和报表导出使用同一套用户数据范围。
- A5 与物料外部交互通过明确的适配器接口完成，支持 `mock` 和 `http` 两种模式。
- 模拟接口提供固定、可重复的数据集，能够验证成功、空数据、超时、错误响应和重复事件。
- 看板、明细与导出基于相同查询条件和数据范围，聚合结果可以互相核对。

### 2.2 非目标

- 本轮不确定真实 A5、仓储或供应商系统的最终字段名称和认证协议。
- 本轮不引入消息总线、服务网格或新的基础设施组件。
- 本轮不重做现有前端页面布局，仅补充必要的模式标识、错误提示和验收入口。
- 本轮不改变现有物料状态语义；若需要增加新的业务状态，应另行评审。

## 3. 总体架构

系统保留现有 FastAPI、SQLAlchemy、Redis/Celery 和 Vue 架构，在服务层增加两个外部适配边界：

```text
前端页面
  -> FastAPI 权限与数据范围
    -> A5IntegrationService -> A5Adapter(mock/http)
    -> MaterialService      -> MaterialExternalAdapter(mock/http)
    -> StatisticsService    -> 统一范围内的数据库事实与集成结果
      -> 看板 / Excel / Word
```

适配器只负责外部协议和数据标准化，状态流转、数量约束、权限判断和统计计算继续由现有领域服务负责。`mock` 模式不访问网络，`http` 模式缺少地址、密钥或令牌时不允许在生产环境静默降级。

## 4. A5 回调安全设计

### 4.1 请求头规范化

签名校验入口首先将请求头名称统一转换为小写，后续只读取：

- `x-a5-signature`
- `x-a5-timestamp`
- `x-a5-event-id`
- `x-forwarded-for` 或 `x-real-ip`

签名原文定义为 `timestamp + "." + raw_body`，算法固定为 HMAC-SHA256。使用 `hmac.compare_digest` 比较签名，不在日志中输出完整期望签名、密钥或请求体。

### 4.2 生产环境失败关闭

- `prod` 环境启动或处理回调时发现 A5 回调密钥为空，必须拒绝服务并给出明确配置错误。
- `local` 和 `test` 环境允许使用专用的非空测试密钥，但不允许“只检查签名头存在”的宽松模式。
- HTTP 适配模式还必须配置基础地址和 API 凭据；生产环境不自动回退到模拟模式。

### 4.3 时间窗与防重放

- 回调时间戳与服务器 UTC 时间偏差不得超过 300 秒。
- 优先使用 `x-a5-event-id` 作为事件键；没有事件 ID 时使用规范化请求体的 SHA-256 摘要生成稳定事件键。
- 相同事件键和相同载荷重复到达时返回已有结果，不重复修改工单。
- 相同事件键但载荷摘要不同，返回冲突错误并记录安全审计日志。

### 4.4 未匹配事件

找不到本地作业工单时，不再只返回“未匹配”后丢弃事件。事件应以 `PENDING_REVIEW` 状态持久化，保留事件键、作业编号、载荷摘要、接收时间、尝试次数和最近错误，供后续人工处理或重试。

## 5. A5 适配器与模拟接口

定义统一的 `A5Adapter` 协议：

```python
class A5Adapter(Protocol):
    async def fetch_daily_reports(self, report_date: date) -> list[A5DailyReport]: ...
    async def fetch_anomalies(self, report_date: date) -> list[A5Anomaly]: ...
    async def fetch_process_progress(self, report_date: date) -> list[A5ProcessProgress]: ...
```

实现两种适配器：

- `HttpA5Adapter`：封装现有 A5 HTTP 客户端，负责认证、超时、状态码映射和外部字段清洗。
- `MockA5Adapter`：根据日期和固定场景返回确定性数据，不访问网络。

模拟数据至少包含：

- 一个从 `DISPATCHED` 推进至 `WORKING` 的日报。
- 一个可归类的施工异常。
- 一个特殊工序进度记录。
- 一个无法匹配本地工单的事件。
- 一个重复事件及一个同键不同载荷事件。
- 可配置的空数据、超时和外部 500 错误场景。

适配器模式由显式配置 `A5_ADAPTER_MODE=mock|http` 控制。本地与测试默认 `mock`，生产仅允许 `http`。

## 6. 物料外部适配器设计

现有物料数据库记录仍是系统内业务真相。新增 `MaterialExternalAdapter` 仅负责与仓储或供应系统交换计划、出库、到场和异常事件：

```python
class MaterialExternalAdapter(Protocol):
    async def submit_plan(self, request: MaterialPlanRequest) -> MaterialPlanResult: ...
    async def fetch_events(self, since: datetime) -> list[MaterialExternalEvent]: ...
```

实现：

- `MockMaterialExternalAdapter`：生成固定计划编号、仓库、供应队伍、出库数量、到场数量和异常记录。
- `HttpMaterialExternalAdapter`：保留正式 HTTP 契约边界；真实字段映射集中放在适配器内，不渗入 CRUD 或统计服务。

外部事件使用 `source_platform + external_event_id` 幂等。事件只调用现有合法状态转换和数量校验，不允许绕过状态机直接更新数据库字段。超时或错误不改变当前物料状态，只记录集成失败并允许重试。

模式由 `MATERIAL_ADAPTER_MODE=mock|http` 控制。本地与测试默认 `mock`；生产选择 `http` 时必须配置地址和令牌。

## 7. 数据范围设计

数据范围在 API 边界解析为不可变的 `DataScope`，然后传递到所有查询服务：

```python
class DataScope(BaseModel):
    is_global: bool
    user_id: int
    department: str | None
    reporting_units: tuple[str, ...]
```

统一规则：

- 超级管理员和明确拥有全局权限的角色可以查看全部数据。
- 普通用户只能查看其填报单位或部门范围内的数据；创建人限制沿用现有项目池规则。
- 物料详情、修改、删除和外部事件处理先按范围查询；范围外资源统一返回 404，避免泄露存在性。
- 物料统计必须在数据库查询阶段应用范围条件，不能先聚合全量数据再过滤。
- 综合统计、Excel 和 Word 必须接收同一个 `DataScope` 和规范化查询对象。
- 统计缓存键必须包含规范化查询和数据范围摘要，避免不同用户共享越权缓存。

## 8. 统计一致性设计

建立一个规范化 `StatisticsAnalysisQuery`，由 API 边界负责日期、枚举和空字符串归一化。统计服务使用同一查询和数据范围构建一次结果，供以下输出复用：

- 统计看板。
- A5 与物料明细下钻。
- Excel 报表。
- Word 报表。

至少增加以下可核对指标：

- A5 同步成功数、失败数、待复核数和成功率。
- 物料需求总数、到场数、使用数、异常数和使用率。
- 作业工单总数、完成数和完成率。

指标定义固定分子、分母、日期字段和空数据结果；百分比只在最终展示时四舍五入。看板总数必须等于相同条件下明细总数，导出值必须等于看板值。

## 9. 错误处理与可观测性

统一外部错误分类：

- `CONFIGURATION_ERROR`：生产环境配置缺失，不重试。
- `AUTHENTICATION_ERROR`：外部认证失败，不自动无限重试。
- `TIMEOUT` 和 `REMOTE_UNAVAILABLE`：按有限指数退避重试。
- `PAYLOAD_INVALID`：记录原始事件摘要并进入人工复核。
- `IDEMPOTENCY_CONFLICT`：拒绝处理并写安全审计日志。

日志必须包含 `trace_id`、适配器模式、事件键、业务编号和错误分类，不记录密钥、令牌、完整签名或敏感原始载荷。同步状态页面展示最近成功时间、最近失败时间、待重试数和待复核数。

## 10. 前端行为

- A5 集成页明确展示当前模式为“模拟接口”或“正式接口”。
- 模拟模式提供场景选择：正常、空数据、超时、错误和重复事件。
- 物料页展示外部来源、外部事件编号和最近同步结果，但所有业务状态变更仍通过受控操作完成。
- 统计页在加载失败时保留上一次成功结果并显示错误状态，不把外部接口失败误报为零数据。
- 数据范围外资源统一展示“不存在或无权访问”，不显示内部权限规则。

## 11. 测试策略

### 11.1 单元测试

- 请求头大小写、正确签名、错误签名、过期时间戳、缺少密钥。
- 重复事件、同键不同载荷、无事件 ID 的稳定摘要。
- A5 和物料 mock 适配器的所有固定场景。
- 物料合法流转、非法跳转、数量上限和外部事件幂等。
- 数据范围谓词、范围外详情和统计缓存键隔离。

### 11.2 集成测试

- 使用临时 SQLite 数据库和 FastAPI 测试客户端执行 A5 回调、物料外部事件、统计查询与导出。
- 使用两个不同填报单位用户证明列表、详情、统计和导出都不会跨范围泄露。
- 使用固定场景证明 A5、物料和作业指标在看板、明细、Excel、Word 中一致。

### 11.3 前端与构建测试

- 契约测试验证模式标识、错误状态、模拟场景操作和统计字段。
- 执行 TypeScript 检查和生产构建。
- 保留大包体积警告为非阻塞项；若本轮修改导致包体继续明显增长，再拆分按路由加载。

## 12. 验收标准

- 小写 `x-a5-signature` 携带正确 HMAC 时回调成功，错误签名和过期时间戳被拒绝。
- 生产环境缺少 A5 密钥、正式接口地址或物料接口令牌时不能静默运行正式适配器。
- 同一 A5 或物料事件重复投递只产生一次业务变更；冲突载荷被拒绝并可审计。
- 非全局用户无法通过列表、详情、统计或导出读取其他填报单位数据。
- 本地环境无需真实外部系统即可完成 A5 日报、异常、工序及物料计划、出库、到场、异常联调。
- 固定验收场景下，看板总数、明细总数、Excel 和 Word 指标一致。
- 聚焦测试、完整后端测试和前端生产构建通过，工作区不产生数据库、日志或密钥文件变更。

## 13. 实施顺序与回滚边界

实施按四个独立阶段推进：

1. A5 回调安全和幂等持久化。
2. 数据范围覆盖物料、统计与导出。
3. A5/物料适配器及模拟场景。
4. 统一统计、前端状态和端到端验收。

每个阶段独立提交并保持数据库迁移可逆。出现问题时优先按阶段回滚；模拟适配器与正式适配器使用相同领域接口，因此切换模式不需要修改业务数据。
